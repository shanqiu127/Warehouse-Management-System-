# 普通员工业务知识库

## 1. 角色定位

普通员工是系统中的执行角色，没有管理菜单和侧边栏，主要通过员工工作台处理工作要求、维护个人信息、查看公告和消息。

## 2. 员工首页（工作台）

员工登录后进入工作台首页，这是员工的核心工作界面。

### 2.1 个人信息区

- 显示员工姓名、所属部门、部门场景描述
- 显示当前登录时间

### 2.2 指标卡片

- 当前部门：显示所属部门名称
- 公告数量：当前可见公告数
- 采购/销售/仓储部门员工额外显示：低库存预警数、零库存预警数
- 人事/财务部门员工额外显示：档案状态（手机号+邮箱+岗位均已填则"已完善"，否则"待完善"）、岗位名称（未完善时显示"未完善"）

### 2.3 我的档案面板

- 只读信息：员工编号、岗位名称
- 可编辑信息：手机号、邮箱
- 如果联系方式未完善，会直接显示编辑表单
- 如果已完善，显示联系信息并提供"修改"按钮
- 手机号格式要求：6-20 位，支持数字、+、-、()、空格（正则 `/^[0-9+\-()\s]{6,20}$/`）
- 邮箱格式要求：标准邮箱格式（正则 `/^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$/`）

### 2.4 部门联系信息

显示所属部门名称、部门负责人、联系电话（只读）。

### 2.5 工作要求列表

按优先级排列显示所有工作要求：超时任务优先、已完成和已拒收排最后。

每条工作要求以卡片形式展示：
- 工作内容（截断显示）
- 超时状态标签（超时中/逾期提交，非正常时才显示）
- 任务状态标签（未接受/已接受/待审核/已完成/已拒收/已驳回）
- 截止时间（格式"截止：YYYY-MM-DD HH:mm:ss"）
- 卡片边框颜色：绿色 = 已完成，红色 = 已拒收，红色背景 = 超时中

点击任务卡片跳转到 /work-requirement/:assignId 详情页。

### 2.6 最新公告

展示当前角色和部门可见的最新公告列表，每条显示标题、受众标签、发布时间。受众标签规则：all → "全员公告"、admin + 部门名 → "部门名 管理员"、employee + 部门名 → "部门名 员工"。点击查看公告详情（弹窗宽 620px，展示标题/受众/发布时间/发布人/内容）。

### 2.7 浮动提醒

- 待接受提醒："你当前有待接受的工作要求，请及时处理。"（信息提示条，可关闭）
- 超时提醒："你当前有超时中的工作要求，请优先处理并补交执行结果。"（红色提示条，可关闭）
- 关闭后本次登录会话内不再弹出。关闭签名存储于 sessionStorage，key 格式为 `employee-{type}-reminder:{userId}:{token}`，签名为 `{userId}:{sortedAssignIds}`。当任务列表变化（如新增任务）签名改变，提醒再次出现

## 3. 工作要求详情页

员工通过此页面处理具体的工作要求任务。

### 3.1 任务信息展示

显示工作内容、开始时间、截止时间、当前状态、驳回次数、超时状态、首次超时时间。

### 3.2 进度步骤条

接受任务 → 提交执行结果 → 审核通过，每一步显示时间。

### 3.3 超时提醒

- 红色警告：当前任务已超时，请尽快补充执行结果并提交
- 黄色警告：该任务为逾期完成或提交已属于逾期提交

### 3.4 任务状态流程

| 状态 | 说明 | 可用操作 |
|---|---|---|
| 未接受 | 收到任务等待处理 | 接受、拒收 |
| 已接受 | 接受任务进入执行阶段 | 填写执行结果并提交 |
| 待审核 | 已提交等待管理员审核 | 无（等待中） |
| 已完成 | 管理员审核通过 | 无（已结束） |
| 已拒收 | 员工拒绝接受该任务 | 无（已结束） |
| 已驳回 | 管理员驳回执行结果需重新提交 | 重新填写执行结果并提交 |

### 3.5 提交执行结果

- 填写执行结果（textarea，4 行，必填）
- 上传附件（图片格式 jpg/png/gif/webp/bmp，最多 5 张，上传方式 picture-card）
- 上传接口 `/api/upload/image`，携带 Authorization Bearer Token
- 提交后进入待审核状态，等待管理员审核
- 被驳回后可再次编辑并重新提交，已有附件保留，可添加/删除

### 3.6 完成状态展示

审核通过后显示：审核人、审核时间、完成口径（按时完成/逾期完成），以及提交的执行内容和附件。

## 4. 站内消息中心

员工可以在页面顶部的消息中心查看系统消息。

### 4.1 消息功能

- 查看未读消息数量（徽章显示）
- 查看消息列表
- 标记已读
- 一键标记全部已读
- 删除全部已读消息

### 4.2 消息类型

- 待接受提醒：管理员发布工作要求后通知
- 超时提醒：任务超过截止时间后通知
- 系统消息：其他系统级通知

## 5. 部门场景说明

员工首页会根据所属部门显示不同的场景描述：

| 部门 | 场景标题 | 场景说明 |
|---|---|---|
| 人事部 | 人事资料与日常协同看板 | 聚焦档案完整性、资料时效和日常组织协同 |
| 采购部 | 采购执行与补货协同看板 | 优先处理补货节奏、供应商跟进和退货登记规范 |
| 销售部 | 销售订单与缺货沟通看板 | 围绕订单核对、库存反馈和退货登记保持闭环 |
| 仓储部 | 仓储预警与盘点纪律看板 | 持续关注低库存、零库存和盘点差异处理时效 |
| 财务部 | 财务对账与月结节奏看板 | 围绕资料齐套、数据核对和月结进度保持准确性 |

## 6. 明确边界

- 没有管理员侧边栏菜单
- 不能进入任何管理员业务页面
- 不能发布工作要求（只能接收和执行）
- 不能发布公告
- 只能处理分配给自己的工作要求任务
- 只能查看针对自己角色和部门的公告
