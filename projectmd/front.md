# 仓库管理系统前端总结（front）

## 一、项目结构与技术栈

| 维度 | 内容 |
|---|---|
| 技术栈 | Vue 3、Vite、Element Plus、Pinia、Vue Router、Axios、ECharts |
| 核心目录 | `src/views`、`src/views/home/components`、`src/api`、`src/router`、`src/stores`、`src/utils`、`src/layout` |
| 工程入口 | `src/main.js`、`src/App.vue` |
| 联调关键文件 | `src/utils/request.js`、`vite.config.js` |
| 权限实现 | 路由守卫 + `v-permission` 指令 + 后端鉴权协同 |

## 二、当前阶段概述



### 1. 当前前端范围
- 已覆盖：登录注册、角色首页、系统管理、基础资料、四类业务单据、库存预警中心、销售统计图表、超管中心、工作要求模块、站内邮箱、首页提醒体系、项目 AI 助手页面与首页悬浮入口。
- 不在本文范围：后端实现细节与数据库脚本说明，见 back.md 与 project.md。

## 三、当前前端实现（核心）

| 模块 | 当前能力 | 关键文件 | 说明 |
|---|---|---|---|
| 登录态与权限上下文 | 持久化 `token/role/deptId/deptCode/deptName/username/realName` | `src/utils/auth.js`、`src/stores/user.js` | 所有路由、菜单、按钮与页面行为均基于该上下文计算 |
| 注册链路 | 注册页动态加载部门下拉，并提交 `deptId` | `src/views/RegisterView.vue`、`src/api/user.js` | 与后端 `/auth/depts`、`/auth/register` 契约对齐 |
| 路由守卫 | 同时校验角色与部门能力 | `src/router/index.js` | `meta.roles + meta.deptCodes` 成为页面访问唯一入口约束 |
| 按钮级权限 | 指令支持角色数组与 `{ roles, deptCodes }` 双写法 | `src/utils/permission.js` | 页面无需继续散落手写角色判断 |
| 角色首页 | `HomeView` 统一分发到 `SuperAdminHome / AdminHome / EmployeeHome` | `src/views/HomeView.vue`、`src/views/home/components/*` | 三类首页已统一为同一视觉语言，但内容按角色与部门分化 |
| 工作要求模块 | 管理员工作要求列表页 + 员工工作要求详情页 | `src/views/system/WorkRequirementView.vue`、`src/views/WorkRequirementDetailView.vue`、`src/api/workRequirement.js` | 管理员走“通知 -> 工作要求”，员工从首页卡片或提醒进入 |
| AI 助手页 | 提供项目问答、历史会话、模型选择与快捷提问 | `src/views/system/ProjectAssistantView.vue`、`src/api/projectAssistant.js`、`src/api/aiConversation.js` | superadmin 与部门管理员可切换模型，employee 仅使用系统默认问答模式 |
| 首页悬浮助手 | 从首页快速拉起 AI 助手入口 | `src/components/assistant/AssistantLauncher.vue`、`src/views/home/components/*` | 作为轻量入口补足“先问再找页面”的使用路径 |
| 首页提醒 | 员工待接受提醒、员工超时提醒、管理员待审核提醒、管理员超时提醒、仓储审批提醒 | `src/views/home/components/EmployeeHome.vue`、`src/views/home/components/AdminHome.vue` | 提醒关闭状态按当前 token 写入 `sessionStorage`，仅对当前登录会话生效 |
| 站内邮箱 | 已登录用户统一显示邮箱入口与未读数 | `src/components/MessageCenter.vue`、`src/api/message.js` | 员工也可接收并查看工作要求超时消息 |
| 系统管理 | 公告、用户、部门、员工页面按角色与部门差异化展示 | `src/views/system/*` | superadmin 具备完整筛选；部门管理员看到收口版表单与表格 |
| 基础资料 | 供应商、商品资料仅仓储管理员可管理 | `src/views/base/*` | 与仓储菜单、按钮权限和后端接口边界对齐 |
| 库存预警中心 | 仓储/采购/销售管理员共用同一页面 | `src/views/business/StockWarningView.vue` | 采购/销售可访问且不再误触发供应商权限提示 |
| 四类业务单据 | 进货、进货退货、销售、销售退货支持查询、建单、详情、删除、历史审批流 | `src/views/business/*` | 历史单据作废/红冲不再直接执行，而是提交仓储审批 |
| 审批状态展示 | 统一使用 `bizDocumentState.js` 解释“已删除/待审批/驳回/作废成功/作废并红冲成功” | `src/utils/bizDocumentState.js` | 第一轮以独立状态列展示，二次迭代改为操作列彩色文本 |
| 统计图表 | 销售/毛利双视角，利润分析仅财务管理员可见 | `src/views/business/SalesChartView.vue` | 与后端财务部门权限口径一致 |

## 四、当前角色与菜单矩阵

### 1. superadmin
- 首页
- 公告管理
- 用户管理
- 超管总览
- 项目 AI 助手
- 安全策略 / 登录日志 / 操作日志

### 2. 部门管理员（role=admin，按 deptCode 细分）
- hr：首页、全部门管理、全员工管理、通知（工作要求/公告管理）、用户部门管理
- purchase：首页、商品进货、进货退货、预警中心、通知（工作要求/公告管理）、项目 AI 助手、用户部门管理
- sales：首页、商品销售、销售退货、预警中心、通知（工作要求/公告管理）、项目 AI 助手、用户部门管理
- warehouse：首页、供应商管理、商品资料管理、预警中心、作废审批、通知（工作要求/公告管理）、项目 AI 助手、用户部门管理
- finance：首页、销售统计图表、通知（工作要求/公告管理）、项目 AI 助手、用户部门管理

### 3. 部门员工（role=employee）
- 当前不再显示左侧侧边栏，员工登录后直接进入单页工作台。
- 首页已升级为“单页多部门员工工作台”，统一展示欢迎区、部门关注指标、我的档案、部门联络信息、工作要求、最新公告与提醒卡片。
- 员工可在首页直接维护手机号和邮箱；若已完善，则展示已保存联系方式卡片并支持点击“修改”进入编辑态。
- 员工可通过首页悬浮助手或独立页面进入项目 AI 助手，但仅允许项目内问题问答，不展示管理侧模型切换能力。

## 五、关键页面与交互口径

### 1. 首页
- `AdminHome.vue`：
  - 人事、财务管理员显示 3 个指标卡：当前部门、公告数量、上次登录。
  - 采购、销售、仓储管理员显示 5 个指标卡：当前部门、公告数量、低库存预警、零库存预警、上次登录。
  - 二次迭代后，第一行仅保留当前登录时间值，不再显示“本次登录/上次登录”文案。
  - 当前已增加右下角提醒堆栈：工作要求待审核提醒、工作要求超时提醒，以及仓储管理员的作废/红冲审批提醒。
- `EmployeeHome.vue`：
  - 已升级为单页多部门员工工作台，统一接入欢迎区、部门化指标、档案信息、部门联络信息、工作要求卡片、最新公告。
  - 欢迎区仅保留当前登录时间，并按部门编码切换工作台标题与副标题。
  - 采购、销售、仓储展示低库存与零库存指标；人事、财务展示档案状态与岗位名称。
  - 联系方式区域支持“已保存信息卡 / 编辑表单”切换，保存后即时回显。
  - 工作要求卡片按超时优先、完成/拒收下沉的顺序展示；待接受提醒与超时提醒卡片都只对当前登录会话生效。
  - 页面已做移动端单列适配，员工角色布局层不再渲染左侧菜单。
- `SuperAdminHome.vue`：
  - 保留数据库状态、24 小时系统错误与最近错误日志。
  - 二次迭代后，数据库状态下移到第二行绿色卡片，错误卡片保持红色。

### 1.1 工作要求页面
- `WorkRequirementView.vue`：管理员侧列表、详情、发布、审核集中页。支持状态筛选、关键词搜索、超时筛选、全员/指定员工发布、员工执行情况表、附件图片预览、审核通过/驳回。
- `WorkRequirementDetailView.vue`：员工侧详情页。支持接受、拒收、填写执行结果、上传图片、提交审核，并在超时场景显示“超时中/逾期提交/逾期完成”提示。
- 管理员详情中已增加“超时状态”“首次超时”列；管理员列表已增加“超时中人数”“逾期提交人数”统计。

### 1.2 项目 AI 助手页面
- `ProjectAssistantView.vue` 提供会话列表、消息流、推荐问题、空状态引导和发送区域，承担前端问答主界面。
- superadmin 与部门管理员可见模型下拉，仅保留模型选择本身，不再展示“首选模型 / 自动回退”提示文案；employee 不暴露模型切换。
- 页面会结合当前登录身份显示不同提问口径，引导员工聚焦项目流程、制度、页面入口、业务规则等项目内部问题。
- `AssistantLauncher.vue` 作为首页悬浮入口，降低进入路径成本，适合“先提问，再跳转定位页面”的使用场景。

### 2. 公告管理页
- superadmin 可按标题、受众角色、目标部门筛选，并可创建管理员公告或全员公告。
- 部门管理员仅能管理本部门员工公告，界面上不再暴露多余受众与部门选择。
- 已移除旧的状态筛选栏与状态表格列，查询、重置、新增统一放在顶部同一行。

### 3. 用户管理页
- superadmin：保留用户名、角色、部门等完整筛选与表单能力。
- 部门管理员：仅保留用户名、状态查询与新增用户；角色和所属部门字段在查询、表格、表单中均已隐藏。
- 交互上只维护“本部门员工账号”，不再暴露越权入口。

### 4. 部门管理页
- 人事管理员可见顶部“新增部门”按钮。
- 表格表头改为“负责人（可多位）”，表单占位提示支持逗号或顿号分隔多个负责人。

### 5. 业务单据页
- 四类单据都遵循：当天单据可删除，历史单据只能申请作废/作废并红冲。
- 删除后，前端当前列表保留记录并标记为“已删除”。
- 历史审批结果会回显到列表中：待审批、驳回、作废成功、作废并红冲成功。
- 二次迭代后，独立“流转状态”列已删除，只在操作列以彩色文本保留状态结果。

### 6. 预警中心
- 当前由仓储、采购、销售管理员共享访问。
- 仓储管理员显示“仓储预警工作台”标签；采购/销售不显示该标签。
- 供应商筛选仅对仓储管理员显示与加载，采购/销售进入页面不会再因请求供应商下拉而弹出无关权限提示。

## 六、前端工程与联调要点

- 开发命令：`npm run dev`
- 构建命令：`npm run build`
- Vite 代理：`/api -> http://localhost:8080`
- 鉴权头：Bearer Token
- 图表加载：ECharts 按需注册 + 动态导入，已做 vendor 拆包
- 图片上传：员工提交工作要求执行结果时通过 `/api/upload/image` 上传，前端统一对 `/uploads/**` 做 `/api` 前缀归一化。
- AI 助手接口拆分为问答与会话两组：`src/api/projectAssistant.js` 负责提问与模型列表，`src/api/aiConversation.js` 负责历史会话查询、消息加载与删除等动作。
